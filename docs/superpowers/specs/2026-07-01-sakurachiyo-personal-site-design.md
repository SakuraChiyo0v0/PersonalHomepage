# SakuraChiyo 个人网站设计规格

## 项目目标

为 SakuraChiyo 构建一个 PC 端优先的个人网站。网站首先呈现 SakuraChiyo 这个人；创作、技术、虚拟世界、朋友和照片只是自然散落其中的生活线索，不作为需要反复解释的身份标签。整体需要轻松、友好，并带有适量二次元与极客气息。首版为单页个人空间，未来可扩展独立画廊和朋友照片页面。

## 受众与语言

- 访客包括开发者同行、潜在合作者、VRChat 朋友和希望进一步认识 SakuraChiyo 的新朋友。
- 页面采用中英混排：展示型标题、标签和终端命令使用英文，介绍性内容使用中文。
- 文案气质自然、温暖、有好奇心，避免企业官网式的严肃表达。

## 视觉方向

选定的设计方向为「漂浮日记终端」。

- 奶油白作为背景色，樱花粉和薄荷青作为点缀色，深墨色负责主要文字和极客感。
- 使用半透明漂浮卡片、细边框、柔和阴影、圆润几何形状和等宽终端字体。
- 二次元元素以漫画网点、花瓣、贴纸、抽象角色剪影和小型装饰符号呈现。
- 首版不依赖受版权保护的动漫素材；抽象角色区域未来可以替换为 SakuraChiyo 的 VRChat Avatar 照片。
- 相比参考图降低视觉密度，使页面保持松弛、清晰和耐看。

## 信息架构

首版采用单页滚动结构：

1. **Hero / Entrance**：SakuraChiyo 名称、双重身份短句、动态终端提示、在线状态和主要操作入口。
2. **About / Two Sides**：Coding 与 VRChat 两张身份卡。
3. **Selected Works**：三张代表性项目卡片与技术标签。
4. **Now / Life Log**：正在进行的事情、近期兴趣和 VRChat 状态。
5. **Memory Preview**：小型拍立得照片带，用于建立未来画廊的视觉方向。
6. **Contact / Find Me**：GitHub、VRChat 和邮箱等联系方式。

主导航使用 About、Work、Moments 和 Contact 锚点。未来增加 Gallery 和 Friends 独立路由时无需重构首页。

## 首屏循环身份卡组

- 删除人物卡与人物视差，首屏右侧完全用于 VRChat、Steam 与 GitHub 三张横向信息卡。
- 正面卡约 `520 × 220px`，使用 14～16px 正文；两张后排卡缩放至约 92%，分别向左右下方错开，只露出平台标题、状态和边缘色。
- 点击任意后排卡时，该卡先向侧面抽出，再沿弧线进入正面；原正面卡同步缩小、下沉并排到卡组末尾。三张卡保持循环排序，不做简单两两交换。
- 动画使用约 650ms 的 GSAP Timeline，同步位置、旋转、缩放、阴影和 z-index。单次切换期间锁定新的切换请求，避免连续输入破坏顺序。
- 卡片本体仅负责切换；正面卡内部提供独立“访问资料”链接，避免轮换与跳转冲突。
- 卡片支持 Enter / Space 键切换；减少动态效果模式下直接更新层级，不执行弧线运动。
- VRChat 卡采用粉紫玻璃质感，展示 `SakuraChiyo`、静态状态签名、常去世界提示与资料页入口。首版不声称提供实时在线信息。
- Steam 卡采用深蓝掌机界面质感，展示静态最近游玩、近两周时长和游戏数量占位，以及真实资料页入口 `76561199038682501`。
- GitHub 卡采用浅墨色终端质感，渐进增强请求 `SakuraChiyo0v0` 的官方公共 API，展示头像、公开仓库数、Followers 与最近更新仓库；请求失败时使用稳定的本地占位内容。
- 窄屏下三张信息卡转为可横向滚动的卡片列表，不保留立体循环运动。
- 音乐播放器移动到右下角，默认仅露出小唱片，展开时向左上方扩展，避免遮挡三张身份卡。

## 交互与动效

动效使用 GSAP，并通过 React 安全的生命周期机制完成清理。

- 首屏分层进入、终端逐字输入和在线状态灯呼吸效果。
- 背景网格与装饰贴纸根据鼠标位置产生轻微视差。
- 两张身份卡随滚动错位展开。
- 项目胶片根据纵向滚动进行横向移动。
- 拍立得照片卡片在滚动时从堆叠状态依次散开。
- 部分桌面按钮提供磁吸、悬浮和轻微倾斜反馈。
- 动画仅使用 transform 和 opacity，避免频繁触发布局计算。
- 当系统启用 `prefers-reduced-motion` 时，关闭滚动绑定、视差、逐字和磁吸效果，同时保留全部内容与导航能力。

动效用于强化内容层级，不能延迟用户访问内容。即使动画不可用，核心交互也必须正常工作。

## 边缘音乐播放器

- 播放器固定在窗口右侧中部，桌面端默认只露出唱片边缘与播放状态，鼠标移入、点击或键盘聚焦时展开。
- 支持播放与暂停、上一首、下一首、进度拖动、音量控制和时间显示。
- 曲目来自 `public/music`，由站点清单明确列出，避免浏览器无法枚举静态目录的问题。
- 使用 ID3 标签读取歌曲标题、歌手与 MP3 内嵌封面；缺少标签或封面时使用文件名和默认渐变唱片。
- 播放时唱片持续旋转，暂停后保留当前角度；切歌时更新封面、元数据和进度。
- 浏览器禁止未授权自动播放，因此首次播放必须由访客主动触发。
- 移动端使用底部迷你播放器，点击后展开控制区域。
- 原 CURRENT STATUS 唱片卡改为 `Currently` 今日便签，展示短句与轻量状态信息，不再出现第二套唱片视觉。
- 播放器控制具备键盘焦点、按钮文本与可访问标签；减少动态效果模式下关闭抽屉弹性和唱片旋转。

## 技术架构

- 使用 React 与 Vite。
- 使用 `gsap`、`@gsap/react` 和 ScrollTrigger 构建动画。
- 页面拆分为 `Hero`、`Identity`、`Projects`、`LifeLog`、`Memories` 和 `Contact` 功能组件。
- 文案、项目和状态卡片存放在独立数据模块中，未来替换内容时无需重写组件。
- 使用 CSS 自定义属性统一管理颜色、字体、间距、圆角和阴影。
- 首版针对 PC 展示优化，同时保证窄屏下可读且不会出现横向溢出。
- 首屏以下图片使用固定宽高比和延迟加载。

## 内容假设

目前已知公开 ID 为 `SakuraChiyo`，但尚未提供最终个人简介、项目资料、外部链接、VRChat Avatar 图片或朋友照片。因此首版使用有完成度的示例文案和易于替换的占位链接与内容，不会把示例项目或状态描述成真实事实。

## 可访问性与容错

- 使用语义化页面区域与连续的标题层级。
- 提供清晰的键盘焦点样式，导航可以通过键盘操作。
- 文本对比度以 WCAG AA 为目标。
- 纯装饰元素对辅助技术隐藏。
- 可选图片缺失时显示风格化视觉面板，不破坏布局。
- JavaScript 动画不可用时，所有内容仍保持可见。

## 验收标准

- 生产构建成功完成。
- 页面无控制台错误，PC 端无意外横向溢出。
- 导航锚点与联系入口支持键盘操作。
- 核心 GSAP 动画流畅，并在组件卸载时正确清理。
- 减少动态效果模式下不出现持续动画和滚动绑定动画。
- 在常见 PC 视口尺寸下完成整页浏览器视觉检查。

## 首版不包含

- CMS、登录、上传功能或后端服务。
- 完整的生产级照片画廊和朋友目录。
- VRChat 实时在线状态集成。
- 独立的移动端交互设计，但会提供可用的响应式降级。
